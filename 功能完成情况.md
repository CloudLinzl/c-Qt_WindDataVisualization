## 主要功能算法实现思路

### 1. 数据读取与时间对齐
**核心挑战**：雷达风速数据和云台角度数据来自不同文件，采样频率不同，时间戳不完全匹配。
**实现思路**：
1.  **数据结构**：分别读取两个CSV文件到 std::vector中。
2.  **时间索引**：为了加速查找，将云台角度数据的时间戳建立索引（或保证有序）。
3.  **对齐策略**：以风速数据的时间戳为基准，对每个风速数据点，在角度数据中寻找时间戳最接近的记录（Nearest Neighbor Search）。
4.  **容差控制**：如果最近的时间戳差异超过一定阈值（如1秒），则标记为无效数据，防止错误匹配。

### 2. 离群值检测与修复
**核心挑战**：雷达数据中常含有噪声和离群点，影响可视化效果。
**实现思路**：
1.  **空间连续性假设**：物理世界的风场在短距离内是连续变化的。
2.  **检测算法**：遍历每条射线上的距离门，计算当前点与前后相邻点（i-1 、i+1）平均值的偏差。
3.  **判定标准**：若偏差绝对值超过设定的阈值（threshold），则判定为离群值。
4.  **修复策略**：直接用前后邻居的平均值替换该离群值。

### 3. PPI扫描可视化 (极坐标映射)
**核心挑战**：将极坐标系下的雷达数据（方位角、距离、数值）映射到屏幕直角坐标系。
**实现思路**：

1.  **坐标变换**：
    *   x = center_x + scale * distance * cos(theta)
    *   y = center_y - scale * distance * sin(theta)
    *   theta 需要考虑数据的起始方位角偏移（如数据0°对应地理东偏南15°）。
2.  **扇形绘制**：
    *   不只是绘制点，而是绘制扇形块（Annulus Sector）。
    *   使用QPainterPath，先 arcTo 外圆弧，再 lineTo 内圆弧起始点，再arcTo内圆弧（反向），最后 closeSubpath。
    *   这样可以保证在放大时画面依然平滑，无缝隙。

### 4. 湍流强度 (TI) 计算
**核心挑战**：在离散的时空点上估算湍流强度。
**实现思路**：
1.  **定义**：TI = 标准差 / 平均风速。
2.  **滑动窗口**：
    *   **时间维**：取当前时刻前后各N帧（如窗口大小5，即 `t-2` 到 `t+2`）。
    *   **空间维**：可选，取相邻距离门。
3.  **计算流程**：
    *   收集窗口内的所有有效风速值。
    *   计算平均值 mean 和方差 variance。
    *   计算 TI = sqrt(variance) / mean。
    *   处理 mean 接近0的除零异常情况。

### 5. 交互式视图控制 (缩放/平移)
**核心挑战**：实现类似地图的平滑缩放和平移，且保持鼠标指向位置不变。
**实现思路**：
1.  **变换矩阵**：维护一个全局的变换状态 offset (平移量) 和 scaleFactor (缩放倍数)。
2.  **坐标映射**：
    *   ScreenPos = (LogicPos * scaleFactor) + offset
    *   LogicPos = (ScreenPos - offset) / scaleFactor
3.  **缩放逻辑**：
    *   记录鼠标当前在逻辑坐标系下的位置 P_logic。
    *   更新 scaleFactor。
    *   反推新的 offset，使得 P_logic 映射回屏幕的位置依然是鼠标当前位置。
    *   new_offset = mousePos - P_logic * new_scaleFactor。

### 6. 高清图像合成导出
**核心挑战**：屏幕截图分辨率受限于窗口大小，且需要组合多个组件。
**实现思路**：
1.  **虚拟画布**：不直接截取屏幕 (grab())，防止缩放导致的异常，而是使用 QPixmap 创建一个指定高分辨率（如1200px宽）的虚拟画布。
2.  **独立渲染**：
    *   实现 renderToPixmap(size) 接口，接受目标尺寸。
    *   在该接口内部，临时调整绘图参数（如缩放比例），将完整内容绘制到虚拟画布上。
    *   绘制完成后恢复原有状态。
3.  **垂直拼接**：
    *   创建总高度为 PPI高度 + Profile高度 的大画布。
    *   使用 QPainter 将两张独立渲染的图片绘制到大画布的指定位置。

## 技术实现细节

### 1. 数据加载 (loader.hpp/cpp)

- **WIND_SPEED类**：读取CSV格式的风速数据，包含时间、距离门、径向风速、SNR
- **GIMBAL类**：读取CSV格式的云台角度数据，包含时间、方位角、仰角
- **DATA_TABLE类**：融合两个数据源，按时间戳对齐
- **CSV导出优化**：在导出CSV时添加UTF-8 BOM头 (\xEF\xBB\xBF)，确保Microsoft Excel能正确识别中文和特殊字符，解决乱码问题。

### 2. 数据预处理 (loader.cpp)

- **离群值检测**：比较相邻距离门的风速，差异超过阈值判定为离群值
- **异常值修复**：使用前后邻居的平均值填充离群值和NaN
- **SNR过滤**：记录每条射线上第一个低于阈值的位置

### 3. GUI主窗口 (MainWindow.h/cpp)

- **菜单系统**：文件菜单（打开、加载、退出）、导出菜单（数据、图像）、视图菜单（切换风速/湍流强度）
- **控制面板**：SNR阈值调节、距离范围设置、时间滑块、显示模式切换
- **可视化区域**：PPI热力图 + 双折线图（风速/SNR）
- **状态栏**：文件状态、扫描模式、数据点统计
- **图像导出增强**：
  - **组合导出**：将PPI热力图和下方的折线图组合导出为一张图片。
  - **高分辨率**：使用 renderToPixmap 接口，独立于屏幕显示大小，导出高分辨率（1200px宽度）图片。
  - **智能布局**：垂直排列PPI和折线图，自动对齐，去除多余空白。
  - **自动命名**：默认文件名使用当前数据的时间戳，格式化为 YYYY-MM-DD_HH-MM-SS.png。

### 4. PPI热力图 (PPIWidget.h/cpp)

- **极坐标系统**：以雷达为中心，半径表示距离，角度表示方位角
- **坐标转换**：方位角0°映射为屏幕角度-75°（东偏南15度），顺时针增长（数据角度与地理角度映射修正）。
- **扇形绘制**：使用QPainterPath绘制每个数据点的扇形区域
- **颜色映射**：蓝→青→绿→黄→红的彩虹色谱
- **交互功能**：
  - **缩放**：重写 wheelEvent，支持以鼠标为中心的缩放。
  - **平移**：重写 mousePress/Move/ReleaseEvent，支持视图拖拽。
  - **悬停**：重写 mouseMoveEvent (无按键)，实时计算鼠标位置对应的极坐标（距离、方位角），并在左上角显示对应的数据值（风速/TI）。
- **性能优化**：
  - **双缓冲**：使用 QPixmap 缓存静态背景和扫描图像，仅在数据更新或视图变换时重绘。
  - **按需渲染**：支持 renderToPixmap 用于导出。

### 5. 湍流强度 (TI) 计算

- **算法**：滑动窗口法
- **公式**：TI = σ / ū (标准差 / 平均值)
- **实现**：在时间和空间维度上取窗口（例如时间窗口大小5），计算窗口内风速的标准差和平均值，得出该点的湍流强度。
- **可视化**：共用PPI显示逻辑，使用独立的颜色映射范围（0.0 - 0.5）。

### 6. 剖面图 (ProfileWidget.h/cpp)

- **坐标系**：Y轴表示距离，X轴表示风速/SNR
- **数据点+折线**：绘制数据点并连接成折线
- **UI优化**：数据点大小缩小至1px，提高精细度。
- **导出支持**：实现 renderToPixmap，支持指定尺寸的高清导出。

## 核心算法

### 离群值检测算法

```cpp
for (每条射线的风速数据) {
    for (每个距离门 i) {
        local = (前一个距离门 + 后一个距离门) / 2
        if (abs(当前值 - local) > threshold) {
            当前值 = local  // 修复为平均值
        }
    }
}
```

### 湍流强度计算

```cpp
double mean = sum / count;
double variance = sqSum / count - mean * mean;
double stdDev = sqrt(max(0.0, variance));
return (mean != 0) ? (stdDev / abs(mean)) : 0.0;
```
